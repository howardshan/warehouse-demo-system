## 1. 项目背景与目标

### 1.1 客户画像（2026-07 调研结论）

- **客户**：Southern Star Foods，高置信度为 Southern Star Distributors I, Ltd. 的商号（DBA），Plano, TX。⚠️ 待与客户当面确认实体身份。
- **实质业务**：小型 broadline 食品批发分销商（非狭义"餐厅设备商"），约 45 人、估算年营收 ~$28M，成立于 ~1987 年。
- **运营硬数据**（FMCSA USDOT 498144）：自营仓库 + 自有车队 **21 辆卡车 / 26 名司机**，跨州配送，货类含干货、生鲜、肉类、冷藏食品、饮料、纸制品。
- **客群（推断）**：独立餐厅/小连锁，固定路线每周配送。
- **现状痛点**：现用 ERP 已 20+ 年，功能落后、卡顿报错。仓库+车队是公司命脉，故对 WMS 格外看重。

### 1.2 Demo 目标

1. **赢得合作**：用可运行的 demo 证明我们理解食品分销业务，能力远超其旧系统。
2. **验证范围**：以 WMS + Sales 两模块为切入点，跑通"下单 → 拣货 → 装车 → 配送签收 → 退货"全链路。
3. **不追求完备**：demo 阶段明确"不做清单"（见 §3.3），管理客户预期。

### 1.3 成功标准

- 客户演示会上能流畅走完一条完整业务故事线（见 §7），无报错。
- 客户能直观感受到与旧系统的代差：实时告警看板、移动端作业、批次追溯秒级响应。
- 演示后客户愿意进入正式合同谈判（Discovery 阶段收费立项）。

**结论：不重构，增量开发。** 下一个迁移序号从 `0033` 起。所有新功能遵循现有模式：领域纯函数进 `lib/domain/`，业务护栏下沉数据库触发器/RLS，页面走 Server Actions。

| customers 表 | 路线绑定字段由 WMS 提交迁移 | 定价/order guide 字段由 Sales 提交迁移 |

### 5.2 协作机制

- **迁移序号排队**：在团队群里认领序号（0033、0034…）再动手，避免冲突；一个迁移文件只属于一个模块。
- **接口点（改动需双方过目）**：
  1. `confirmSalesOrder` / `revalidateSoGates` —— 信用/毛利/ATP 三道闸（Sales 调用，WMS 提供 ATP）；
  2. SO 冻结点 → 拣货单生成（`generatePickList`）；
  3. 分客户定价影响 SO 价格快照列，不得破坏"价格快照"铁律（invariants）；
  4. cut-off/路线绑定影响 SO 可选配送日（WMS 提供，Sales 消费）。
- **共同纪律**：改 `lib/domain/`、`docs/00-invariants.md`、RLS/触发器前先在群里说一声；`npm run docs:check` 与 `vitest` 过了才能合。
- **Git**：git 历史目前不可用（5 个 commit 全叫 "001"）。demo 期起强制规范：feature 分支 + 有意义的 commit message，建议尽快建远程仓库。

## 6. 技术方案要点

- 沿用现有架构，**零新框架引入**：Next.js Server Actions 即后端，业务护栏下沉 Postgres 触发器/RLS，领域纯函数进 `lib/domain/` 并配 vitest。
- 移动端不做原生 App：响应式 Web（手机浏览器），扫码用 Web API（`BarcodeDetector`，降级为手输）。正式合同阶段再评估 Flutter（公司既有能力）。
- 新页面继续走三语 `messages/*.json`（西语对德州仓库/司机是实用卖点，演示时可现场切换）。
- 新表一律带 RLS + 审计（沿用 audit_log 模式）。

## 7. 演示策略：一条订单的一生

演示会故事线（约 25 分钟，双人配合）：

1. **销售**（同事）：餐厅客户来电 → order guide 快速下单 → 信用闸拦截超额客户 → 审批放行
2. **仓库**（本人）：cut-off 后订单落入周二线路 → 生成拣货单 → 手机扫码拣选（含一条 catch-weight 肉类称重）→ 倒序装车清单
3. **司机**（本人，用真手机）：逐站配送 → 一站拒收两箱 → 现场签名 POD → 拒收自动进退货/credit 队列
4. **管理层视角**：dashboard 实时告警 + 任选一个批次演示"3 秒双向追溯"（对比旧系统卡顿报错，此处是情绪高点）
5. 收尾：明确"今天没演示的"（§3.3 不做清单）→ 引出正式合作的 Discovery 提案
