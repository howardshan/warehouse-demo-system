# ADR-0004: 信用占用为什么必须含「已签收未开票」

## 状态
已采纳

## 背景
本系统**不做开票**。传统 ERP 的 AR 占用依赖 invoice。

## 问题
若 exposure 只算已开票未付款 → 永远是 0 → 信用控制静默失效（铁律 5）。

## 决策
```
exposure = 已签收未开票 + 已开票未付款(未来) + 可选(已确认未发货)
```
Phase 1：客户信用字段 + 仅 finance 可改 status。  
Phase 6：`v_credit_exposure`。

## 后果
✅ 不开票也能控信用  
❌ 签字质量直接影响占用；必须 POD 闭环
