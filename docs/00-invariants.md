# 14 条业务铁律（不可违反）

> 每一条都必须在**数据库层**用约束 / 触发器强制，不能只靠 UI。
> 绕过任意一条，系统会**静默出错**，几个月后才爆炸。
> Phases 1–9 已完成；下列 Phase 标注表示铁律首次落地位置。

## 铁律 1：成交价必须物理存储，不得引用主档

- **规则：** `so_lines.unit_price` / `cost_snapshot` 是真实列；建单时抄入。
- **不做的后果：** 主档改价后，历史订单金额被改写 → 发票错 → 客户不认。
- **上线时症状：** 无。几个月后查历史才发现。
- **Phase：** 1 主档铺垫；4 落 SO 快照（已完成）。

## 铁律 2：价格锁在「行」上，不锁在「单头」上

- **规则：** 同一张 SO 可有不同单价的行（T+1，拣货前可加行）。
- **不做的后果：** 后加行被错误套用建单时整单价格，或无法表达行级折价。
- **Phase：** 4（已完成）。

## 铁律 3：数量一律双单位；重量列必须 `_lb`

- **规则：** `qty_units` + `qty_weight`/`*_lb`；ATP 按件，计价按重；**严禁用均重填实重**。
- **不做的后果：** 称重废弛；召回/开票金额错；单位 kg/lb 混淆。
- **Phase：** 1 商品字段；2–8 贯穿交易全链路（已完成）。

## 铁律 4：状态机是护栏

- **规则：** `pending_approval`/`credit_hold` 不能生成拣货单；漏称不能装车；签字后 SL 锁死等。
- **不做的后果：** UI 禁用可被 API 绕过 → 违规流转。
- **Phase：** 4–6（已完成）。

## 铁律 5：信用占用不能等 invoice

- **规则：** exposure 主体 = 已签收未开票；本系统不做开票。
- **不做的后果：** exposure 永远 0，信用控制静默失效。
- **Phase：** 1 客户字段；6 落占用视图（已完成）。

## 铁律 6：信用要查两次

- **规则：** SO 确认时 + **装车放行前**。
- **不做的后果：** 货上车后才发现超限，收不回来。
- **Phase：** 4 + 6；Phase 1 信用状态写护栏（已完成）。

## 铁律 7：FEFO 在补货时控制

- **规则：** 拣货位同时只能一个批号；补货按效期最早。
- **不做的后果：** 批号混堆 → 召回客户名单错误。
- **Phase：** 1 储位类型；3 触发器（已完成）。

## 铁律 8：SO 冻结点 = 拣货单生成

- **规则：** 不是「开始抓货」。要改必须先显式撤回拣货单。
- **不做的后果：** 拣货员手里的纸已成废纸而不自知。
- **Phase：** 5（已完成）。

## 铁律 9：改单必须重跑三道闸

- **规则：** 信用 / 毛利 / ATP，每次编辑后重跑。
- **不做的后果：** 加行超限/缺货/低毛利漏检。
- **Phase：** 4（已完成）。

## 铁律 10：无欠单 — 关单释放预留

- **规则：** SL 签字关 SO；未发 `allocated_units` 归零。
- **不做的后果：** 幽灵预留吃光 ATP，「有货显示无货」。
- **Phase：** 6（已完成）。

## 铁律 11：食品退货默认报损，回库换新批号

- **规则：** 先进隔离区；可回库必须新批号 + parent 映射。
- **不做的后果：** 追溯链污染。
- **Phase：** 7（已完成）。

## 铁律 12：批号追溯链不可断

- **规则：** 分装/退货母子映射持久化；支持按批号查下游客户。
- **不做的后果：** 食安召回失败。
- **Phase：** 8（已完成）。

## 铁律 13：收货必须盲收

- **规则：** UI 不显示 PO 数量；三单核对不可合并。
- **不做的后果：** 收货员照单填数，失去互相印证。
- **Phase：** 2（已完成）。

## 铁律 14：有差异必须选枚举原因

- **规则：** `variance_reason` 必填枚举；`stock_mismatch` 触发盘点。
- **不做的后果：** 库存不准被掩盖，仓库管理水平失真。
- **Phase：** 5（已完成）。
